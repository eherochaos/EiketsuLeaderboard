# Visual Acceptance

## 桌面 `1280x720`
- 首页必须看到 Header。
- 首页必须看到环境数据标题。
- 首页必须看到更新时间。
- 首页必须看到主导势力。
- 首页必须看到 TierList CTA。
- 首页必须看到当前第一卡组。
- 首页必须看到 Top3 卡组。
- 首页状态栏顶部 `32px` 必须出现在首屏内。
- TierList 必须看到标题区。
- TierList 必须看到筛选区。
- TierList 必须看到表头或首行。
- 代表卡组 8 卡构成与指标区禁止重叠。

## 手机 `390x844`
- 首页必须单栏堆叠。
- 首页必须无页面级横向溢出。
- 首页必须看到标题、更新时间、CTA、当前第一。
- 势力占比必须显示最多 7 个官方非零分段和 5 个刻度。
- 代表卡组必须显示 4x2 卡组盘。
- 代表卡组按钮文本必须完整显示。
- 注目单卡必须显示行块。
- TierList 必须显示行块。
- TierList 行块必须显示 Rank、卡组名、指标、依据标签。
- 手机禁止依赖横向表格。

## 对齐
- 主容器必须居中。
- Header 内容必须与主容器对齐。
- Section 左边界 x 坐标必须一致。
- 代表卡组每行指标区必须纵向对齐。
- 代表卡组按钮必须纵向对齐。
- 表格数字列必须使用 `text-align: right`。
- 卡图必须等距。
- 卡槽必须等高。

## 间距
- Section 间距必须为 `24px`。
- 卡片内部 padding 必须一致。
- 标题到说明必须为 `6px`。
- 说明到内容必须为 `16px`。
- 列表行距必须为 `12px`。
- 连续空白禁止超过 `48px`。

## 色彩
- 页面背景必须使用 `--color-bg`。
- 顶部导航必须使用 `--color-brown`。
- 主 CTA 必须使用 `--color-red`。
- Active 背景必须使用 `--color-gold-fill`。
- 数据图颜色只能使用 7 个官方势力 token。
- 势力色必须核对 `design-tokens.md` 的 7 个官方 token。
- 蒼必须是 `#1e3ca0`。
- 緋必须是 `#b4191b`。
- 碧必须是 `#007332`。
- 玄必须是 `#636261`。
- 紫必须是 `#8c0078`。
- 琥必须是 `#ff7800`。
- 黄必须是 `#c9a227`。
- 禁止黑蓝科幻感。
- 禁止全页面单一色相。

## 文案审计
- 页面禁止出现人工标题。
- 页面禁止出现人工理由。
- 页面禁止出现人工影响说明。
- 页面禁止出现 `reason`。
- 页面禁止出现 `description`。
- 页面禁止出现内部组件名。
- 页面禁止出现真实 API 暗示。

## 文本适配
- 按钮文字必须完整显示。
- Select 长文本必须省略且不撑破控件。
- 卡组名长文本最多显示两行。
- 表格名称长文本最多显示两行。
- 指标数字禁止换行。
- 长文本禁止造成页面级横向溢出。
- 禁止用缩小字号解决长文本。

## 数据审计
- Top3 合计必须等于 `factionShare` 前三项之和。
- 主导势力必须等于 `factionShare` 前三项。
- TierList 初始第一必须为最高 `rankScore`。
- 筛选后桌面表格和移动行块必须同步。
- 注目单卡标签必须符合阈值。

## 技术验收
- 控制台必须 0 JS 错误。
- 图片加载失败时原图片容器宽高必须不变。
- 所有图片必须有 `alt`。
- 首页状态切换必须正常。
- TierList 筛选必须正常。
- TierList 排序必须正常。
- 页面禁止出现横向溢出。
