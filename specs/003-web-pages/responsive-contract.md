# Responsive Contract

## Breakpoints
- Desktop 必须为 `1100px` 及以上。
- Tablet 必须为 `761px` 到 `1099px`。
- Mobile 必须为 `760px` 及以下。
- Compact Mobile 只能用于 `430px` 及以下的局部修正。
- 字号禁止使用 `clamp()`。
- 字号禁止使用 `vw`、`vh`、`vmin`、`vmax`。

## Desktop
- 主容器最大宽度必须为 `1180px`。
- 页面左右 padding 必须为 `16px`。
- Section 间距必须为 `24px`。
- Section padding 必须为 `22px 24px`。
- H1 必须为 `52px`。
- Section 标题必须为 `28px`。
- 卡组名必须为 `21px`。
- 当前第一卡组名必须为 `25px`。
- 正文必须为 `16px`。
- 表格正文必须为 `15px`。
- 代表卡组必须使用 6 列横向情报行。
- 代表卡组 8 卡构成必须与指标区同排。
- TierList 必须显示桌面表格。

## Tablet
- 页面左右 padding 必须为 `14px`。
- Section 间距必须为 `20px`。
- Section padding 必须为 `20px`。
- H1 必须为 `42px`。
- Section 标题必须为 `25px`。
- 卡组名必须为 `19px`。
- 当前第一卡组名必须为 `23px`。
- 正文必须为 `16px`。
- 表格正文必须为 `15px`。
- Hero 必须切换为单列。
- 代表卡组必须使用上方信息行加下方 8 卡构成。
- 代表卡组 8 卡构成必须独占整行。
- TierList 必须显示桌面表格。

## Mobile
- 页面左右 padding 必须为 `12px`。
- Section 间距必须为 `16px`。
- Section padding 必须为 `16px`。
- H1 必须为 `34px`。
- Section 标题必须为 `22px`。
- 卡组名必须为 `18px`。
- 当前第一卡组名必须为 `20px`。
- 正文必须为 `15px`。
- 表格正文必须为 `14px`。
- Hero 必须单列堆叠。
- 状态栏必须单列堆叠。
- 势力占比必须保留横向堆叠条。
- 代表卡组必须使用信息区加 4x2 卡组盘。
- 注目单卡必须使用行块。
- TierList 必须使用行块。
- Mobile 禁止依赖横向表格。

## Compact Mobile
- Top3 行头像必须为 `44px`。
- Top3 行 Rank 列必须为 `34px`。
- Top3 行分数列必须为 `42px`。
- Compact Mobile 禁止新增页面级横向滚动。
