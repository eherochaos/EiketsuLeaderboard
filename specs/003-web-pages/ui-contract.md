# UI Contract

## 核心铁律
- UI 禁止露出内部组件名。
- UI 禁止露出接口名。
- UI 禁止露出验收稿用语。
- 禁止展示人工标题。
- 禁止展示人工理由。
- 禁止展示人工影响说明。
- 禁止新增必须人工维护的 `reason` 字段。
- 禁止新增必须人工维护的 `description` 字段。
- 禁止新增必须人工维护的 `editorNote` 字段。
- Hero 标题必须由 `updatedAt` 或统计区间生成。
- 主导势力必须由 `factionShare` 排序生成。
- Top3 合计必须由 `factionShare` 前三项计算。
- 卡组排序依据必须由指标生成。
- 单卡指标依据必须由指标标签生成。
- 详情入口只能跳转或占位。
- 禁止定义详情页内容。

## 首屏信息密度
- `1280x720` 首页首屏必须看到环境结论。
- `1280x720` 首页首屏必须看到当前第一卡组。
- `1280x720` 首页首屏必须看到 Top3 卡组。
- `1280x720` 首页首屏必须看到 TierList CTA。
- `390x844` 首页首屏必须看到标题。
- `390x844` 首页首屏必须看到更新时间。
- `390x844` 首页首屏必须看到 TierList CTA。
- `390x844` 首页首屏必须看到当前第一卡组。
- Hero 桌面最小高度必须使用 `--hero-min-height-desktop`。
- `1280x720` 首页首屏必须露出状态栏顶部 `32px`。
- 连续无信息空白禁止超过 `48px`。

## 页面结构
- 首页必须按 Header、Hero、状态栏、势力占比、代表卡组、注目单卡、字段说明排列。
- TierList 必须按 Header、标题区、筛选排序栏、榜单主体、详情入口、字段说明排列。
- Header 只能保留品牌和一个 `.nav-primary` 入口。
- 首页 `.nav-primary` 必须是 TierList。
- TierList `.nav-primary` 必须回首页。
- 其它入口只能使用文字链接样式。

## 视觉方向
- 页面背景必须使用 `--color-bg`。
- 顶部导航必须使用 `--color-brown`。
- 主 CTA 必须使用 `--color-red`。
- 强调元素必须使用 `--color-gold`。
- 内容台背景必须使用 `--color-panel`。
- 禁止大面积纯白。
- 禁止大面积黑底。
- 禁止厚重黑金边框。
- 禁止复杂炫光。
- 禁止科幻蓝黑图表。
- 禁止所有模块使用相同黑底标题条。

## 文案规则
- 页面文案必须是产品语言。
- 页面文案只能说明状态、指标、入口、字段来源。
- 标题禁止由人手命名环境结论。
- 摘要禁止写主观判断。
- 说明禁止要求编辑补内容。
- 字段说明只能说明字段来源。
- 空状态只能说明当前无数据。
- 加载状态禁止展示假数据。

## 图片规则
- 卡图必须使用真实外链或文本失败态。
- 图片必须有 `alt`。
- 图片失败态必须保留原图片容器宽高。
- 禁止下载官方图片到仓库。
- 禁止使用几何图形冒充卡图。
- 禁止使用抽象 SVG 作为主视觉。
- 禁止声明外链素材为项目自有素材。
