# Page Patterns

## 首页 Header
- Header 必须使用 `.top-shell`。
- 导航内容必须居中到 `1180px`。
- 左侧必须显示品牌。
- 品牌符号必须使用 `--color-gold` 菱形。
- 右侧只能有 TierList `.nav-primary`。
- Header 底部边框必须为 `1px solid rgba(198, 146, 51, 0.72)`。

## 首页 Hero
- Hero 必须使用两栏布局。
- 左栏必须展示环境数据概览。
- 左栏必须展示生成标题。
- 左栏必须展示自动生成摘要。
- 左栏必须展示更新时间。
- 左栏必须展示主导势力。
- 左栏必须展示 TierList CTA。
- 左栏目录入口必须使用文字链接样式。
- 右栏必须展示当前第一卡组。
- 右栏必须展示 Top3 卡组。
- 主视觉 opacity 必须为 `0.05`。
- 主视觉必须使用 `position: absolute`。
- 主视觉禁止参与 Hero grid 布局。

## 首页状态栏
- 状态栏必须位于 Hero 下方。
- 状态栏背景必须使用 `--color-panel`。
- 状态按钮必须包含正常、过热中、无数据、缺乏新构筑。
- Active 状态背景必须使用 `--color-gold-fill`。
- 静态筛选项必须位于状态按钮之后。
- 静态筛选项禁止使用 Active 状态。
- 状态栏控件必须设置 `appearance: none`。

## 势力占比
- 势力占比必须是单一横向堆叠条。
- 堆叠条必须显示五段。
- 每段必须显示势力名。
- 每段必须显示百分比。
- 下方必须显示 `0/25/50/75/100%` 刻度。
- 下方代表条目必须来自 `factionShare.representatives`。
- 禁止在堆叠条右侧渲染额外榜单。
- 禁止饼图。
- 禁止环图。
- 禁止图像扇区。

## 代表卡组
- 代表卡组必须是 4 条横向情报行。
- 每行左侧必须显示 Rank。
- 每行左侧必须显示主卡头像。
- 每行正文必须显示势力标签。
- 每行正文必须显示卡组名。
- 每行正文必须显示指标依据标签。
- 每行中部必须显示核心构成。
- 每行核心构成必须支持 8 张卡。
- 每行右侧必须固定展示胜率和使用率。
- 每行右侧必须显示查看构筑按钮。
- 所有查看构筑按钮必须统一样式。

## 注目单卡
- 注目单卡桌面表格背景必须使用 `--color-panel`。
- 表格必须有 Rank、素材、单卡、使用率、胜率、综合 Rank 趋势、指标依据。
- 综合 Rank 趋势必须使用 `78px x 36px` 静态 SVG 折线。
- 指标依据必须使用标签。
- 移动端必须切换为行块。
- 移动端行块必须显示卡图、Rank、名称、指标、趋势。

## TierList Header
- TierList 标题区 padding 必须为 `26px 0 16px`。
- 标题区必须显示榜单名称。
- 标题区必须显示更新时间。
- 标题区必须显示样本数说明。
- 标题区必须显示当前第一。
- 标题区目录入口必须使用文字链接样式。
- 标题区禁止大面积海报。

## TierList 筛选栏
- 筛选栏必须紧贴标题区。
- 筛选项必须包含势力。
- 筛选项必须包含命名来源。
- 筛选项必须包含排序。
- Select 背景必须使用 `--color-panel`。
- Select 边框必须使用 `--color-panel-border`。
- 筛选栏禁止撑高首屏。

## TierList 榜单
- 桌面必须使用表格。
- 移动端必须使用行块。
- 初始排序必须按综合 Rank 降序。
- 表头或首行必须出现在 `1280x720` 首屏内。
- 行内必须显示指标依据。
- 行内禁止展示人工理由。

## 字段说明
- 字段说明必须位于页面底部。
- 字段说明只能解释字段来源。
- 字段说明禁止写未来规划。
- 字段说明禁止声明真实接口存在。
