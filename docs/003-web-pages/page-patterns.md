# 页面结构

- 页面必须使用 Header + Main。
- Main 必须使用统一主容器。
- 首页必须按环境概览、当前第一、Top 3、势力占比、注目单卡排列。
- 首页首屏必须回答当前环境由谁主导。
- TierList 必须按筛选栏、排序状态、榜单表格、分页入口排列。
- TierList 默认必须按综合 Rank 排序。
- TierList 行必须展示卡组名、分类、Rank、胜率、使用率、样本数。
- 详情页必须先展示关键结论。
- 详情页只能展示已有字段。
- 搜索页必须先展示输入框和结果状态。
- 列表页必须保留 loading、empty、error 状态。
- 数据页必须先给结论，再给明细。
- 图表区必须靠近对应数据表。
- 筛选栏必须靠近被筛选内容。
- 操作按钮必须靠近操作对象。
- 桌面端可以使用两栏。
- 1024 以下必须收敛为一栏。
- 390 宽度必须无横向滚动。
- Footer 只能放低优先级链接。
- 禁止把核心数据放进 Footer。
