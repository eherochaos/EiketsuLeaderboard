# 卡组分类 v3

- 必须声明分类是本项目算法。
- 禁止宣称官方分类。
- 必须按 `deck_fingerprint` 识别精确卡组。
- 必须优先判断多枚数。
- 必须将 7 枚以上归为 `多枚数`。
- 必须将 6 枚且低 Cost 多的卡组归为 `多枚数`。
- 必须用每局计略发动频率选择主轴。
- 必须在频率接近时优先 Cost 高的主轴。
- 必须在频率接近时标记 `needsReview`。
- 必须将全体强化归为 `号令`。
- 必须将阵型归为 `号令`。
- 必须将单体强化归为 `バランス`。
- 必须将伤害技归为 `バランス`。
- 必须将未知计略类型归为 `バランス` 并复核。
- 必须输出 `主轴卡名 + 类型 + デッキ`。
- 禁止用固定搭配生成默认分类名。
- 必须把固定搭配放入证据。
- 必须把真实数据作为 CLI 参数传入。
- 禁止把真实数据复制进仓库。
