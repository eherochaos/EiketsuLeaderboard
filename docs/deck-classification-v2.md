# 卡组分类 v3

- 必须声明分类是本项目算法。
- 禁止宣称官方分类。
- 必须按 `deck_fingerprint` 识别精确卡组。
- 必须优先判断多枚数。
- 7 枚以上必须归为 `多枚数`。
- 6 枚且低 Cost 多必须归为 `多枚数`。
- 必须用每局计略发动频率选择主轴。
- 频率接近时必须优先 Cost 高并复核。
- `バランス` 中号令频率达 70% 且每局 0.8 必须升级主轴。
- 全体强化和阵型必须归为 `号令`。
- 单体强化和伤害技必须归为 `バランス`。
- 未知计略类型必须归为 `バランス` 并复核。
- 必须输出 `主轴卡名 + 类型 + デッキ`。
- 证据足够时必须输出 `主轴や第二轴 + 类型 + デッキ`。
- 第二轴必须拆分 `categoryId`。
- 第二轴只能从非主轴卡选择。
- `号令` 第二轴只能接受 `号令` 或 `陣形`。
- `バランス` 第二轴禁止接受 `号令` 和 `陣形`。
- 多枚数禁止输出第二轴名称。
- 禁止低 Cost 通用组件成为第二轴。
- 类型冲突禁止进入分类名。
- 固定搭配只能放入证据。
- 固定搭配禁止等同第二轴。
- 真实数据必须作为 CLI 参数传入。
- 禁止复制真实数据进仓库。
- 传入 Registry 时必须保留历史分类。
- 本次未命中历史分类必须标记 `inactive`。
- 分类改名必须保留旧名到 `aliases`。
